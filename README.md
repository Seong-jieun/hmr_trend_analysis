## 분석 내용
### 데이터 전처리
- raw data의 컬럼이 `대분류/중분류/소분류`로 구분되어 있고, 중분류는 동일한 내용이 일정한 개수에 따라 반복됨
  - ➡️ `대분류_소분류` 형식으로 컬럼명 통일
- 최상위 구분을 `8개`에서 `5개` 분류로 축소
    - `거주, 행정구역, 가구원, 성별, 연령, 학력, 직업, 소득` ➡️ `거주, 가구원, 성별, 연령, 소득`
- 2018-2020년 데이터와 2021-2022년 데이터 형식에 차이가 있어 concat 과정에서 문제 발생
  - `2018-2020년 데이터 항목` ➡️ `2021-2022년 데이터 항목`의 이름으로 변경
- 데이터 형식이 바뀌면서 존재하지 않던 컬럼이 생성되거나 삭제되는 현상 발생
    - 2018-2020년 데이터보다 2021-2022년 데이터의 컬럼 수가 줄어듦
        -  필요한 컬럼의 값들의 평균값으로 파생데이터 생성
            - ex) 4인/5인이상 ➡️ 4인이상 = (4인 + 5인이상) / 2
      - 2021-2022년 데이터의 형식으로 항목별 컬럼명 수정
          - ex) 소스류, 양념류 ➡️ 소스/양념류

## 회고 및 개선점
### 회고
- 통계 데이터를 활용하다보니 데이터의 값이 비율로 되어있어 의미 해석이 어려운 부분이 있었고 파생 데이터를 생성할 때 제한적이었다
- 가구원 수 분류를 1인, 2인, 3인, 4인 이상으로 분류하려 했으나 2-3인으로 묶인 형태가 있었고 이를 임의로 2인, 3인으로 가공할 수 없었다
- 지역과 관련된 의미있는 인사이트 도출을 하지 못했다

### 개선점
- 최신 데이터를 사용하여 현재 트렌드를 파악하고 향후 예측해 볼 수 있다
- 공공데이터가 아닌 크롤링, API 활용 등 데이터를 다양하게 수집하여 분석해 볼 수 있다
- 간편식 구매처 데이터를 활용하여 온라인, 오프라인 중 어디에서 구매가 이루어지는지 분석해 볼 수 있다
